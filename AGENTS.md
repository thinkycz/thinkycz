# AGENTS.md

This document contains essential information about the `thinkycz` project directory, intended for AI agents and developers to quickly understand the project's architecture, tools, and conventions.

## Project Overview

- **Framework**: [Jigsaw](https://jigsaw.tighten.co/) (Static Site Generator by Tighten)
- **Templating**: Laravel Blade (`.blade.php`)
- **Frontend Build Tool**: [Vite](https://vitejs.dev/) (Migrated from Webpack/Laravel Mix)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/) (v3) + Sass (`.scss`)
- **Deployment**: GitHub Pages (via `gh-pages` branch)

## Directory Structure
- `source/`: Contains all the uncompiled static site content.
  - `_layouts/`: Contains Blade layout templates (e.g., `master.blade.php`).
  - `_assets/`: Source directory for assets built by Vite:
    - `js/main.js`: Main JavaScript entry point.
    - `sass/main.scss`: Main Sass entry point.
  - `index.blade.php`: The English (default) homepage.
  - `cs/`: Czech language pages.
  - `vi/`: Vietnamese language pages.
  - `assets/`: Static assets (fonts, images, etc.) that are copied to the build directory.
- `build_local/` & `build_production/`: The compiled, static files generated by Jigsaw/Vite. **Do not edit files here directly**.
- `tasks/`: Contains old Webpack/BrowserSync build scripts. This project now uses Vite exclusively via `package.json` scripts, so these files are legacy.

## Key Configuration Files
- `config.php`: Jigsaw configuration file (defines build environments, collections, baseUrl).
- `vite.config.js`: Vite configuration, utilizes `@tighten/jigsaw-vite-plugin` to orchestrate JS/CSS builds and trigger Jigsaw builds.
- `tailwind.config.js`: Tailwind CSS configuration. Notably, it contains a custom script to statically generate and safelist the *entire color palette* (excluding deprecated colors) to ensure color utility classes like `bg-red-500` aren't purged even if constructed dynamically. It also defines custom animations like `blob`, `carousel`, `fade-in`, and `fade-in-up`.
- `package.json`: Contains the npm scripts and frontend dependencies.

## Standard Development Workflow

1. **Local Development**:
   ```bash
   npm run local
   ```
   Or `npm run dev`. This spins up the Vite dev server with Jigsaw auto-reloading.

2. **Building for Production**:
   ```bash
   npm run build
   ```
   Compiles assets for production and generates the static HTML in the `build_production/` folder.

3. **Deployment**:
   ```bash
   npm run deploy
   ```
   This command automatically runs `vite build`, then pushes the contents of the `build_production/` directory to the `gh-pages` branch using `git subtree push`.

## Architectural Notes
- **Multi-language**: The site has 3 languages implemented statically in the `source/` folder (`en` at root, `cs` and `vi` in subdirectories).
- **Vite Migration**: This project has been migrated from Webpack. Rely on the Vite workflow instead of any legacy `task/` logic.
- **Tailwind Colors**: Due to dynamic usage, most Tailwind colors are globally safelisted inside `tailwind.config.js`. Avoid regex-based safelists for colors, as the custom JS generation script already handles it.
